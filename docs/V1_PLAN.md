# V1 Implementation Plan

Minimal end-to-end implementation of the chunk storage and rendering architecture. No LODs, no culling, no greedy meshing. The goal is to get voxels on screen with the new architecture.

## chunk -- Full implementation (without LODs)

- `BlockId` (`u32`), `ChunkStorage` (Filled/Paletted), `ChunkData` (`Arc<ChunkStorage>`)
- `ChunkMap` (`HashMap<IVec3, Entity>` resource)
- `ChunkPos` component
- `ChunkChanged` event
- Palette compression with variable bit width
- Voxel get/set operations on `ChunkStorage`

## chunk::loading -- Mock loader

- Loads a fixed 8x8x8 area of chunks around the origin
- Spawns entities with `ChunkPos` + `NeedsGeneration`, inserts into `ChunkMap`
- No unloading, no dynamic loading based on camera position

## chunk::generation -- Simple terrain

- Sin-based height map (similar to current implementation)
- Queries `With<NeedsGeneration>`, produces `ChunkData`, fires `ChunkChanged`
- Synchronous for v1 (async can be added later without changing the interface)

## chunk::meshing -- 1x1 faces only

- Change resolver: consumes `ChunkChanged`, adds `NeedsRemesh` to affected chunk + neighbors
- Face extraction: 6 `[BlockId; 32^3]` intermediate buffers, neighbor boundary checks
- No greedy meshing -- directly convert 1x1 faces from intermediate buffers to `[Vec<Face>; 6]`
- Adds `ChunkFaces` component to entity
- Synchronous for v1
- Greedy meshing can be inserted between extraction and output later

## render -- Modul integration

### GPU Buffers (Bevy Resources)

Three pre-allocated GPU buffers, created in an `Init` system:

- **Face buffer** -- `MAX_PAGES * PAGE_SIZE * 8 bytes`, vertex + copy_dst
- **Metadata buffer** -- `MAX_PAGES * 16 bytes`, storage + copy_dst
- **Indirect buffer** -- `MAX_PAGES * 16 bytes`, indirect + copy_dst

### Page Allocator (Bevy Resource)

Simple free list tracking which pages are in use. Manages allocation/deallocation of page indices. Each chunk entity stores its allocated page indices as a component (`ChunkPages`).

### Bind Groups (via modul SimpleBindGroupLayoutBuilder)

- **Group 0**: camera uniform -- `add_uniform::<Mat4x4Uniform>("view_proj")`
- **Group 1**: metadata storage buffer -- `add_entry("metadata", ...)` for the page metadata buffer

### Pipeline (via modul GenericRenderPipelineDescriptor + RenderPipelineManager)

- Vertex state: two `Uint8x4` attributes (face data), instance-stepped
- Fragment state: basic solid color or material-based
- Depth test enabled
- Pipeline cached via `RenderPipelineManager`, auto-adapts to surface format

### Synchronize Systems (run in `Synchronize` schedule set, before draw)

1. **Page upload system** -- queries entities with `ChunkFaces`:
   - For each direction's face vec, split into pages of 512
   - Allocate page indices from free list
   - `queue.write_buffer` face data and metadata to GPU at page offsets
   - Add `ChunkPages` component to entity, remove `ChunkFaces`
2. **Camera uniform system** -- updates view_proj matrix

### Render Operation (via modul OperationBuilder / Operation)

```
VoxelRenderOpBuilder:
    writing() -> [surface_target]
    finish()  -> VoxelRenderOp

VoxelRenderOp::run():
    1. Build indirect buffer: one draw arg per active page (no culling for v1)
       - vertex_count: 6
       - instance_count: face count in page
       - first_vertex: 0
       - first_instance: page_index * PAGE_SIZE
    2. queue.write_buffer the indirect buffer
    3. begin_ending_pass on surface target
    4. Set pipeline, bind groups (0: camera, 1: metadata)
    5. Set face buffer as vertex buffer
    6. multi_draw_indirect
```

### Sequence

Built in `Init`:
1. `ClearNext` -- clear color + depth
2. `VoxelRenderOpBuilder` -- the draw operation

Added to `RunningSequenceQueue`.

### Shader (WGSL)

- Vertex: reads face data from vertex attributes, page metadata from storage buffer via `instance_index / PAGE_SIZE`. Computes world position from chunk origin + local pos. Expands instance to quad using vertex_index (0-5).
- Fragment: basic lighting (directional, normal from direction field in metadata)

Bind group WGSL is auto-generated by modul's `SimpleBindGroupLayoutBuilder` with `#BIND_GROUP` substitution via `PipelineLayoutComposer`.

## camera -- Existing

- Keep current camera controller

## Not in v1

- LODs
- Greedy meshing (architecture supports trivial insertion)
- Frustum / backface culling
- Async generation and meshing
- Chunk unloading
- Block registry
